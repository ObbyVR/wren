Privacy Policy

Effective 2026-04-22 · Wren IDE v0.1.x

Wren is a desktop application that runs locally on your machine. This policy describes the narrow set of data Wren handles and what leaves your device.

1. Short version

  • Your code never touches our servers. Wren does not have servers that receive your project files.
  • Your API keys stay on your device. They are encrypted via the OS keychain and stored at <userData>/wren-keys.json.
  • Provider calls go directly from your machine to Anthropic, OpenAI, Google, or Mistral. Wren is not a proxy.
  • Telemetry is off by default. No automatic metrics, no crash reports, no behavioural analytics unless you opt in under Settings → Telemetry.

2. Data Wren stores locally

  • wren-keys.json — encrypted provider API keys
  • wren-keys-meta.json — non-secret key metadata (label, createdAt, lastUsedAt)
  • wren-snapshots/ — per-project agentic snapshots (retained 30 days, auto-pruned)
  • wren-audit.log — append-only audit events (key set/remove, snapshot rollback). Rotates at 5 MB, 90-day retention.
  • Project configuration and recent-file lists, in the Electron user-data directory

All of the above live under the OS-managed Electron userData folder. Wren never uploads any of it.

3. When data leaves your machine

Data leaves your device only when you explicitly invoke an AI action. In that case, the active provider's SDK sends the request directly to:

  • api.anthropic.com (Anthropic)
  • api.openai.com (OpenAI)
  • generativelanguage.googleapis.com (Google)
  • api.mistral.ai (Mistral)
  • The local Claude Code or OpenAI Codex CLI if you use subscription mode (still local except for the provider call those CLIs make)

Wren receives the streamed response and renders it. No copy is kept on any Wren-owned server.

4. Telemetry (opt-in)

If you opt in under Settings → Telemetry, Wren sends anonymous event counts (commands invoked, errors encountered) to help prioritise roadmap items. It never includes prompts, code, or keys. You can revoke consent at any time; the setting takes effect immediately and nothing is batched.

5. Browser Bridge extension

The optional Chrome / Firefox extension connects to a local WebSocket server started by Wren. All traffic stays on your machine (localhost). The extension reads DOM from pages you explicitly open in the bridged window and never sends that data anywhere except the Wren desktop app running on the same device.

6. Auto-update

When WREN_UPDATE_URL is configured, Wren periodically checks that URL for a new version manifest. No user identifier is attached. You can disable auto-update in Settings.

7. Licensing

License validation is offline. A Pro or Team key carries its tier and expiry signed at issuance time; Wren verifies locally. No license server pings occur.

8. Your rights

Because Wren does not collect personal data on its own infrastructure, the usual GDPR/CCPA access and erasure requests resolve to: delete the app and wipe the Electron user-data folder. If you believe a Wren component is sending data we have not disclosed, email privacy@wren.dev and we will investigate.

9. Changes to this policy

Material changes are announced in the changelog and on the website. The effective date at the top of this page reflects the last update.
